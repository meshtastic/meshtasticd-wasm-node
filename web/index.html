meshtasticd — full node in WebAssembly, radio over WebUSB

Boots the real firmware setup()/loop() in the tab, driving a MeshToad SX1262 over WebUSB.

SDK-driven version · hardware probe · source

idle

Needs Chromium (Chrome/Edge) — WebUSB is Chromium-only. Plug in a CH341-based LoRa adapter (Meshstick / MeshToad / Pine64 hat); on Linux it must not be claimed by a kernel driver, on Windows it needs the WinUSB driver (Zadig). Nothing is sent to any server — the node runs entirely in this tab and the log is the firmware's own stdout. Node identity, config and nodedb persist in IndexedDB across reloads.
